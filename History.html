One vital piece of the World Wide Web that contributed to its development was HTML, or the Hyper Text Markup Language. This language provides the foundation necessary to display content on the web as it provides a structure that “the browser reads the file and translates the text into a visible form” (Hayes, 2025). This structure uses different elements to separate and categorize content based on what they are and dictates how they will be displayed by a browser. This results from multiple decades of development starting back in the early 90’s when Tim Berners-Lee invented the Web.

According to Ragget in the second chapter of Ragget on HTML 4, HTML's core was based on several pre-existing ideas established before its creation, namely Hypertext, the Domain Name System, and the Standard Generalized Mark-up Language. These pieces would finally be fit together in 1991 to create the first version of HTML and would garner some interest, including from Ragget. It is from here that it begins to take off, with more people invested in it, we would see more development with Ragget's HTML+, which expands on the first version, being featured in the first World Wide Web Conference in May of 1994 (Ragget et al., 1998). Soon after in 1994, HTML 2.0 would be introduced, officially adding even more elements, including the development of style sheets and Headings, which are major components that contribute to the web pages we see today.

It is here that we will backtrack slightly and talk about how HTML can be used so far as this is the point that the internet really begins to take shape, and around the time the dot-com bubble forms. At its core, HTML is used to structure web pages using the markup elements called tags and navigate between them using hypertext. In 2.0, it has some of the more basic tags, such as the header tags to mark sections of headings and subheadings, paragraph tags for blocks of text, and the anchor tag used to create hyperlinks and link pages. These and other tags are used together to create interconnected web pages, some of which have begun to be stylized using the style sheets to make them look better or just different from others. From this, companies began to pay much closer attention to the web, as it was clear it was not going away and could further their reach.

Jumping back to its history, we would see HTML’s next major development in 1995 with the first attempt at proposing its next step at HTML 3.0 by Ragget. Although, from what the book says, it was never made official, there was clearly enough support from people at the time that people outside the space dubbed it so. All of the articles I have seen going over its history seem to agree on that point. In the background of these developments, other things were created as a result of HTML development, such as the creation of web browsers such as Lynx and Mosaic, but it was after 3.0 that Microsoft would unveil Internet Explorer. Almost two years later, as a result of collaboration from “… vendors like Microsoft, Netscape and IBM” HTML 3.2 would be approved by the World Wide Web Consortium, which would bring many major changes and features (Jiminez, 2023). HTML 3.2 offered a greater amount of functions, customization, and features than any version before it, allowing people to do things like display videos embedded in a page.

Speeding up a bit in 1999 and 2000, we would see the development of HTML 4.0, 4.01, and XHTML. The former two would update HTML, adding elements, fixing known problems, and improving cross-platform compatibility. The latter, however, was different as XHTML was a combination of the newer Extensible Markup Language (XML), a language that was more flexible and versatile than HTML, with HTML 4.01. From here, things change a little for HTML as in the late 2000’s, HTML 5.0’s first draft was introduced, but a few years later, the W3C and the Web Hypertext Application Technology Working Group decided to do away with the large numbered versions for a single continuously developed version.

Jumping back again, I will be focusing on some changes and developments that happened as a result of HTML’s development. One of those was the adoption of the Cascading Style Sheet, which was a great innovation for styling at the time. HTML has many features that allow a person to control how text and other elements are presented, but in many ways, they were also limited as HTML was strict and did not adapt well across different technologies, and that is where CSS comes in. According to Jay (2017), the greatest benefit of CSS is its simplicity, and instead of defining every little thing, you could instead set rules for the computer to follow when displaying the content. This, of course, changed things for many, as you could reference this style for however many pages you needed and simplify how you create additional pages, and allow pages to adapt better to different technologies, mobile devices would benefit most notably.

Another major thing that resulted from HTML’s evolution was the creation and rise of Media Platforms. Media platforms, especially social media, has existed since essentially the beginning of the web but began to rise rapidly in the 2000’s when more work was being put into other functions of HTML. Video and image sharing was something that could be done beforehand, but after HTML5, it became significantly easier to do as new elements were added to display media natively. Platforms that already existed and shared such media before would quickly adopt it to allow it natively, and eventually, many others Platforms we know today would be created and use it as well.

The final thing I will focus on that was created as a result of HTML was Progressive Web Applications, which is meant to “…offer a consistent experience during wonderful devices and systems (Embracing the Evolution: The Rise of Progressive Web Apps, 2024).” This partially relies on the CSS development as well because they essentially aim to present web pages with an application-like experience. They would offer web-based things like a search function and application functions like notifications and offline availability. There are several examples of PWAs that many people use, like Instagram, YouTube, Outlook, and Spotify. Each of these leverages systems to present content embedded into websites on mobile devices through applications they have downloaded. They have become an essential technology used by many groups and relied on by many people across the world.

HTML’s importance to the current internet can not be understated; it is the backbone that holds most everything up and the core of many of the developments that make it what it is today. Almost forty years after the inception of the web, it is almost unrecognizable when compared to its current state. The first decade alone saw so many changes that it is hard to understand how people kept up with them, with every other year seeming to have another major development or innovation that would change everything. Then there are the things stemming from it, not just the technologies, but other things as well, like companies, governing bodies, events, and products. There is no Item made from a single piece; however, HTML was an incredibly important one which change things for many.
